# Тестирование (skill-assessment)

Документ описывает **стратегию проверки** пакета и сценария **экзамена с протоколом** (веб-чат + дублирование в Telegram-боте), в том числе когда нет полного набора «боевых» телефонов и идентификаторов.

## Продуктовые допущения (для тест-дизайна)

| Тема | Решение |
|------|--------|
| Сценарий | Один: экзамен с протоколом. |
| Каналы | Веб-чат; тот же сценарий дублируется в Telegram-боте. |
| Обратная связь | Транскрипт + текст бота; голос бота — отдельная фаза (v2). |
| Оценка | Балльная шкала (3–4 балла); процент — отдельно (на старте ориентир — `sentence-transformers`; детали позже). Проходного балла нет. |
| Протокол | Сущность в системе + ACL, срок хранения, экспорт PDF. |
| Согласие | При первом обращении к сотруднику — на запись/обработку ПДн и результатов проверки. |

## Уровни тестирования

### 1. Автотесты (без Telegram и без вторых устройств)

- **Юнит-тесты домена**: модуль `skill_assessment.domain.examination_scoring` — косинус по векторам, процент, шкала 0–3, средний процент по сессии; `FixedVectorEmbedder` подставляет фиксированные векторы вместо модели (см. `tests/test_domain_examination_scoring.py`).
- **Интеграция с API**: `TestClient` против приложения (см. [README](README.md)); контекст `with TestClient(app) as c:` — чтобы отработал startup и создались таблицы.
- **Каналы (веб / Telegram)**: исходящие сообщения Part1 идут через **`skill_assessment.adapters.telegram_outbound`**. В pytest по умолчанию **`SKILL_ASSESSMENT_TELEGRAM_OUTBOUND=mock`** (см. `tests/conftest.py`): заглушка пишет «отправки» в память, без реального Bot API.

Пример запуска из venv ядра с установленным `skill_assessment[dev]` (путь к репозиторию подставьте свой):

```powershell
pytest D:\path\to\repo\skill_assessment\tests\test_domain_examination_scoring.py -q
pytest D:\path\to\repo\skill_assessment\tests\test_telegram_outbound_integration.py -q
pytest D:\path\to\repo\skill_assessment\tests\test_assessment_flow.py -q
```

### 2. Локальный и staging-прогон (веб)

- Поднять приложение по инструкции из [README](README.md) (`uvicorn skill_assessment.runner:app`, рабочий каталог — корень клона ядра `typical_infrastructure`).
- Проверять UI и API с **основного ПК**; второго «сотрудника» можно имитировать **вторым браузером** (другой профиль или режим инкогнито), если не критичен именно мобильный UX.

### 3. Ручная проверка Telegram с одним дополнительным устройством

Если нет множества реальных номеров и аккаунтов, достаточно **одного тестового бота** и **одного второго устройства** (например планшет с собственным номером и Telegram):

1. Создать бота через [@BotFather](https://t.me/BotFather), получить токен (только для **staging / dev**, не коммитить в репозиторий).
2. На планшете открыть чат с ботом и отправить `/start` — получить стабильный **chat_id** (через отладочный эндпоинт, лог webhook или временный скрипт; способ зависит от реализации).
3. Сохранить этот `chat_id` в профиле **тестового сотрудника** в системе.
4. Прогнать полный путь: приглашение → согласие → экзамен → протокол; убедиться, что сценарий совпадает с веб-чатом.

Так закрывается риск «мы не видели поведение в реальном мессенджере» без доступа к чужим телефонам.

### 4. SMS и голос

- **SMS**: в разработке часто отключают или используют песочницу провайдера; для сценария экзамена часто достаточно **email + веб + Telegram**.
- **Голос бота**: не входит в текущую фазу тестирования функционала (v2).

## Регрессия и частота

- **CI**: автотесты без внешних каналов — основа регрессии на каждый коммит/PR.
- **Устройство + Telegram**: раз в спринт или перед релизом — smoke по чек-листу (приглашение, согласие, сессия, протокол, ACL/PDF, если реализовано).

## Открытые технические моменты

- Точная формула **процента** и при необходимости **структуризация ответа** перед оценкой — по мере появления в коде дополнять этот документ или соседние спецификации.
- Когда в зависимости появится `sentence-transformers`, имеет смысл зафиксировать в CI отдельную job с кэшем модели или лёгким «smoke»-прогоном, чтобы не раздувать время каждого юнит-прогона.
