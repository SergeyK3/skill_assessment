# Регламенты Word: генерация версии V2

Краткое описание того, как из `docx` в `V1` собираются обновлённые регламенты в `V2`, и откуда берутся навыки и KPI.

## Папки

| Путь | Назначение |
|------|------------|
| `typical_infrastructure/docs/regulations/V1` | Исходные регламенты (не перезаписываются скриптом) |
| `typical_infrastructure/docs/regulations/V2` | Результат: новые версии `docx` |

Относительно корня репозитория `repo`: каталог `typical_infrastructure` лежит на один уровень выше `skill_assessment`.

## Скрипт

**Файл:** `skill_assessment/scripts/build_regulations_v2_from_project_matrices.py`

**Запуск** (из каталога `repo/skill_assessment`, чтобы работали импорты пакета):

```bash
cd repo/skill_assessment
python scripts/build_regulations_v2_from_project_matrices.py
```

Требуется установленный `python-docx`.

## Источники данных (матрицы)

Навыки и KPI подставляются из демо-матриц проекта (те же структуры, что используются для seed и при необходимости для PostgreSQL в Docker):

- `skill_assessment/data/top20_position_skills.py` — `TOP20_POSITION_SKILL_ROWS`
- `skill_assessment/data/top20_position_kpis.py` — `TOP20_POSITION_KPI_ROWS`

Сопоставление кода должности из шапки регламента с матрицей: через поле «Должность» `(CODE)` или через `REG_*_Vn` и алиасы (`DIRECTOR` → `ADM_DIRECTOR`, `SYSADMIN` → `ADM_SYS_ADMIN`, `SALES_MGR` → `SALES_MANAGER`).

## Что делает скрипт

1. **Мета-таблица:** инкремент кода регламента и поля «Версия», дата документа, очистка полей «Основа», подгон ширин колонок.
2. **Дубликат:** удаляется однострочная таблица с блоком «Назначение должности», дублирующим раздел о цели.
3. **Нумерация:** разделы с номером ≥ 7 сдвигаются на +1; затем вставляется раздел **7. Ключевые навыки** с таблицей и текстами-предупреждениями.
4. **Таблица навыков:** колонки №, Навык, Приоритет, Версия (формат совпадает с мета-версией документа).
5. **Таблица KPI:** добавляется колонка «Код и версия»; колонка «Как измеряется» без дублирования кода (код вынесен в новую колонку).
6. **Оформление:** интервалы в ячейках таблиц, ширины колонок, у заголовков разделов вида `N. …` — интервал перед абзацем 12 pt; удаление пустых абзацев в теле документа.

## Константы в скрипте

В начале файла задаются:

- `CURRENT_DATE_STR` — дата в шапке и в подписи версии матрицы в таблицах
- `DOCUMENT_VERSION` и производная `MATRIX_VERSION_LABEL` (например `V2 от ДД.ММ.ГГГГ`)

При смене даты или номера версии правьте эти константы и заново запускайте скрипт.
