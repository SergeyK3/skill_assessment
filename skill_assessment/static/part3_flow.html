Оценка навыков — рабочая сессия

Укажите client_id в URL (из /clients или /client/… → Приложения), чтобы загрузить сотрудников из ядра. Без него используется demo_client и список может быть пустым. Открывайте ядро с тем же хостом и портом, что и API (например http://127.0.0.1:8000 — не http://127.0.0.1 без порта).

Оценок ещё нет — что делать: это ожидаемо в начале. В таблице нажмите Выбрать — список сократится до этого сотрудника. Ниже ведётся один единый цикл оценки для этапов 1–3: система переиспользует активную сессию и создаёт новую только после завершения предыдущей. Если списка сотрудников нет — сначала добавьте их в рабочем пространстве организации.
Протокол опроса по регламентам (сценарий в Telegram после опроса по документам): Открыть список протоколов — HTML по завершённым опросам. В таблице «История назначений сессий» ниже у каждой строки в колонке «Действия» также есть ссылка «Протоколы опросов» (фильтр по сотруднику).

История назначений сессий

Сессии оценки в контексте —. Выберите сотрудника — список сузится к его назначениям. Фильтр «Слот опроса» — по дате/времени слота в Telegram (DOCS_SURVEY_LOCAL_TIMEZONE).

Смена даты/времени слота опроса по документам: в правой колонке у сессии со статусом не completed/cancelled нажмите «Изменить слот» — откроется блок «Шаг 1» ниже с полями даты, времени и кнопкой «Сохранить слот» (тот же запрос, что ручное PATCH …/docs-survey-slot).

Фильтр «Предстоящие»: остаются сессии, у которых время слота в вашей рабочей зоне (как в боте и в полях «Изменить слот» — см. DOCS_SURVEY_LOCAL_TIMEZONE в .env) ещё не наступило. Колонка «Слот опроса» и сохранение слота показывают локальное время с подписью зоны. Создано / обновлено — время записи в базе (вторая строка — последнее изменение сессии); это не то же самое, что время в колонке «Слот» (слот может быть 07:27, а «обновлено» — 07:21, если сохранение было раньше). Если после смены слота строка пропала — в «Слот опроса» выберите Все или Запланированные (есть слот). «Период с / по» — по дате создания сессии, не по дате слота.

1 Опрос по служебным документам

Слот в Telegram сотрудник выбирает в боте; HR может переопределить дату и время здесь: кнопка «Изменить» (ниже) или в таблице «История назначений сессий» — «Изменить слот», затем «Сохранить слот».

Что ожидать после выбора слота: не в момент слота, а примерно за N минут до него (по умолчанию 5, переменная DOCS_SURVEY_REMINDER_MINUTES_BEFORE, максимум 120) в Telegram приходит напоминание с кнопками «готов / не готов». Если сотрудник нажимает «готов», открывается сценарий опроса по регламентам в Telegram; сам «опрос по документам» по длительности привязан к слоту. Если HR переносит слот вручную, напоминание придёт в новое окно; планировщик работает, пока запущен процесс API (uvicorn). Сразу после смены слота отдельного «приглашения» в чат может не быть — ждите окно напоминания за N минут до слота.

После старта сессии активен этап part1: сотруднику уходит уведомление в Telegram. Вопросы по регламенту задаются в опросе в Telegram; формулировки и ответы фиксируются в протоколе опроса (страница «Протоколы опросов» / ссылка из уведомления). На этой странице отдельный чек-лист с дублирующими вопросами не показывается.

Если назначение ошибочное или тестовое, отмените его в блоке «История назначений сессий» (кнопка «Отменить») — статус станет cancelled, сотрудник не будет «застревать» в незавершённой сессии. Чтобы убрать строку из истории полностью (в т.ч. после завершения или отмены), используйте «Удалить» — запись удалится из базы без восстановления.

Если сотрудник ничего не получил — нажмите после выбора сессии. Не для завершённых и отменённых назначений.

Слот опроса по документам (локальное время, зона из DOCS_SURVEY_LOCAL_TIMEZONE):

Для сотрудника используется одна общая сессия оценки. Запустите этап 1 или сохраните слот — при необходимости сессия создастся автоматически.

Без старого чек-листа: фаза сессии оценки переключится на part2.

2 Кейсы и оценка ИИ

Для сотрудника формируется набор из 2–3 кейсов, где каждый кейс может покрывать сразу несколько навыков. Задача набора кейсов — суммарно закрыть весь список навыков; при необходимости HR может дозапросить дополнительные кейсы по непокрытым навыкам.
